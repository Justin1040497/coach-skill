# Coach

`Coach` 是一个面向学习项目、作品集项目、实习项目、校招项目和比赛项目的项目引导型 Skill。

触发后，它会先扫描项目，理解项目内容，拆分功能模块，识别项目架构，然后询问用户接下来要继续推进、整理项目，还是解决具体问题。

## 核心定位

`Coach` 的目标是帮助用户把项目看清楚、拆清楚、讲清楚，再决定怎么做。

它关注：

- 当前项目是什么
- 已经完成了哪些功能
- 功能完成到什么程度
- 项目可以拆成哪些功能模块
- 项目更接近什么架构
- 用户想继续推进、整理，还是修复某个模块
- 模块变更前是否已经想清楚流程和异常情况

## 默认流程

触发后，`Coach` 会：

1. 判断当前目录是否是项目。
2. 如果是项目，直接扫描；如果不是，询问项目路径。
3. 总结项目类型和现有功能。
4. 按功能模块拆分项目。
5. 使用专业名词说明项目架构。
6. 询问用户想继续推进、整理项目，还是解决问题。
7. 在用户确认模块后，先商量方案，而不是直接写代码。
8. 方案清楚后，提供 Mermaid 流程图和文本视觉流程图。
9. 询问用户自己写，还是让 Agent 按确认的流程图写。

## 功能模块拆分

`Coach` 拆的是项目在做什么，而不是代码架构层级。

例如笔记项目可以拆成：

- 编辑器系统
- 图片处理系统
- 撤回重做系统
- 本地存储系统
- 搜索与分类系统

不应该拆成：

- domain 层
- application 层
- infrastructure 层
- repository 接口
- repository 实现

这些架构层可以在架构说明里讲，但不作为功能模块。

## 模块变更流程

当用户要新增、修改或更新某个模块时，`Coach` 会先确认：

- 模块目标
- 用户交互流程
- 输入和输出
- 状态变化
- 数据创建、保存、恢复、删除时机
- 异常和边界情况
- 与其他模块的关系
- 哪些内容不在本次范围内

如果关键细节不清楚，`Coach` 会先问，不会猜。

## 流程图

方案确认后，`Coach` 会同时提供：

- Mermaid 流程图
- 文本视觉流程图

文本视觉流程图会使用缩进、箭头和 `Y/N` 分支，方便快速阅读。

## 执行选择

流程图完成后，`Coach` 会询问：

```text
你希望自己实现，还是由 Agent 按照确认的流程图实现？
```

如果用户自己写：

- 提供实现步骤
- 提供检查点
- 提供验收标准
- 不提供完整代码
- 不直接改文件

如果 Agent 写：

- 严格按照确认的方案和流程图实现
- 不写计划外功能
- 关键细节不清楚就询问
- 语法、框架、库或工具链问题先查官方文档或可靠资料
- 完成后在项目根目录创建 `docs/logs/` 并写入本次完成总结

## 完成日志

Agent 实现完成后，日志至少包含：

- 本次行为总结
- 按照哪份方案或流程图执行
- 修改了哪些文件
- 新增了哪些文件
- 每个新增文件的作用
- 删除了哪些文件
- 如果没有删除文件，明确写“无删除文件”
- 还有哪些未完成或需要用户确认的点
- 验证方式或测试结果

## 最终总结

写完日志后，`Coach` 还需要给用户一段很好懂的最终总结。

这段总结不是日志，也不是单纯列文件。它更像是当前完成的功能总结，用户可以直接拿去做项目介绍或面试答辩。

它要用大白话说明：

- 完成了什么功能或解决了什么问题
- 使用了什么技术方案或架构
- 为什么这个方案适合当前项目
- 实现时考虑了哪些关键情况
- 为什么没有使用另一个方案
- 这个功能大概是怎么工作的
- 如果别人问“这个功能是怎么做的”，用户可以怎么回答

最终总结要尽量白话，不要堆术语。需要用术语时，要马上用简单的话解释。

总结后，`Coach` 还要询问用户是否听懂，并提出几个面试风格的问题。

这些问题可以考察很简单的内容，但表达要稍微专业一点，帮助用户适应面试官常见的问法。例如：

- 为什么选择这种实现方式？
- 这个模块在整体流程里解决了什么问题？
- 你考虑了哪些边界情况？
- 如果执行到一半失败，会发生什么？
- 如果数据量变大或需求变复杂，这个设计要怎么调整？

## 文件结构

- `SKILL.md`：Skill 主规则和路由规则
- `agents/openai.yaml`：展示名、默认提示词等元数据
- `references/planning.md`：项目扫描、功能模块地图和架构识别规则
- `references/coaching.md`：模块方案、流程图、执行选择和 Agent 实现规则
- `references/review-checklist.md`：项目地图、模块方案和完成结果的检查清单
